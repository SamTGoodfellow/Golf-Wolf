4. Functional Requirements
4.1 Game Setup

User can add players:

Name (required)

Handicap (integer)

Minimum of 3 players required to start

Players can be edited or removed before starting

No persistence required (session-based is fine for MVP)

4.2 Hole Management

Game tracks holes from 1 to 18

Current hole number is always clearly visible

User can move to the next hole

Wolf resets when moving to the next hole

4.3 Wolf Selection

For each hole, user can select one player as the Wolf

Wolf should be visually highlighted (icon or colour)

Wolf selection should be quick (single tap)

Only one Wolf allowed per hole

4.4 Scoring (MVP Rules)

For the MVP, keep scoring intentionally simple:

User can mark one or more players as ‚ÄúWon Hole‚Äù

Each win awards +1 point to that player

Scores are tracked per hole and summed into a total

Advanced Wolf rules (solo wolf, doubles, presses, betting) are explicitly out of scope for v1.

4.5 Score Display

Each player shows:

Name

Handicap

Current total score

Scores update immediately after input

Wolf is clearly marked in the player list

4.6 Leaderboard

App shows a leaderboard:

Players sorted by total score (highest first)

Leaderboard is viewable:

During the round

Automatically at the end of hole 18
